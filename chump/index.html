PCB Computer

Computers can often be a black box.

The Why

There are so many systems working to make the picture you see on your screen, it's seems impossible to figure out all that's going on.

The What

This project attempts to strip down a computer to what it was originally: a device capable of executing a program in order to fulfill a given purpose. For the most basic tasks, a computer needs 5 things:

  • a program to run
  • a counter to keep track of the program line
  • a method of decoding the current the operation
  • a chip to carry out the logic operation
  • memory to store and read data

Let's use this Sim!

I've made a little simulation of my project for you to play with right here on this page!

If you've programmed in Assembly, this is like my own personal version.
If not, this will be a new thing to learn :).

With these simple concepts the computer can perform these operations:

  • Load
  • Addition & subtraction
  • Bitwise ANDing
  • Storing and reading to memory
  • Conditional & non-conditional jumping
Each operation is made up of an operation and a constant. The constant will be used in conjuction with the result from the previous operation, shown by the LEDs from the output of the ALU (Arithmetic and logic unit). For memory instructions, the constant is treated as an address, and for the goto or the if-zero conditional, the constant is treated as the line number to jump to.

The program currently running reads address 2 from memory and adds 1 to the value (shown in the accumulator). It then stores that value back and restarts the program by jumping to the first line (0).
As you can see the accumulator is incrementing by one, showing the output of the program!
